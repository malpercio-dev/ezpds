# AGENTS.md — Project Conventions and Dev Environment

## Summary

The root-level project configuration file for the [[concepts/ezpds-workspace|ezpds workspace]]. It defines the full tech stack (Rust stable, Cargo workspace, SQLite via [[entities/sqlx|sqlx]], Nix flake + devenv, just task runner), development commands, and dev environment setup. The file establishes the Functional Core / Imperative Shell architecture pattern enforced across all crates.

Key conventions include: workspace-level dependency versions, no ticket references in source code, mandatory Bruno API collection updates for route changes, and explicit boundaries around files that must never be hand-edited (`flake.lock`, `devenv.local.nix`). The Nix dev shell provides all tools — developers must not install globally.

The file also documents the iOS build toolchain resolution: `ios-env.sh` dynamically resolves the Apple toolchain via `xcode-select`, with host-target overrides gated on `EZPDS_IOS_BUILD=1` so non-iOS builds remain unaffected.

## Key Takeaways

- **Nix-first development**: All tools managed by Nix flake + devenv; direnv auto-activates via `.envrc`. Always run `nix develop` from workspace root.
- **Functional Core / Imperative Shell**: Every `.rs` file with runtime behavior must have a pattern comment. Pure logic in core crates, I/O boundary in the relay.
- **No ticket references in code**: Source code comments describe *why* in system terms, not which ticket required it. Ticket traceability belongs in `docs/` design plans.
- **iOS toolchain is dynamic**: `ios-env.sh` resolves Apple toolchain at build time; host overrides gated on `EZPDS_IOS_BUILD=1` to avoid affecting relay builds.
