# README.md — Project Overview

## Summary

The project README for [[concepts/ezpds|ezpds]], an easy-to-host ATProto Personal Data Server. It describes the three main components: the [[entities/relay|Relay Server]] (Axum-based HTTP server implementing ATProto provisioning API, XRPC endpoints, and OAuth 2.0), the [[entities/crypto|Crypto Library]] (P-256 keys, did:key, did:plc lifecycle, Shamir secret sharing), and the [[entities/identity-wallet|Identity Wallet]] (Tauri v2 iOS app guiding users through account creation, DID ceremony, and identity recovery).

The architecture diagram shows the iOS identity wallet communicating with the relay server for account provisioning, DID ceremony, and OAuth authentication. The relay stores accounts, DIDs, handles, and signing keys in SQLite.

The project status section documents working capabilities (account creation, DID creation, handle registration, OAuth 2.0, password reset, Shamir recovery, OpenTelemetry) and planned waves (blob handling, federation, app proxy, hardening).

## Key Takeaways

- **Three-component architecture**: iOS wallet → Relay server → SQLite. The wallet is the user-facing layer, the relay is the API layer, the crypto crate is the pure-logic layer.
- **ATProto compliance**: Implements provisioning API, XRPC endpoints (`com.atproto.server.*`, `com.atproto.identity.*`), OAuth 2.0 with DPoP, and did:plc lifecycle.
- **Recovery via Shamir**: 2-of-3 threshold secret sharing for DID rotation key recovery, with iCloud Keychain backup for Share 1.
- **Wave-based roadmap**: Current focus on Wave 4 (blobs), then federation (firehose, subscribeRepos), app proxy, and hardening.
