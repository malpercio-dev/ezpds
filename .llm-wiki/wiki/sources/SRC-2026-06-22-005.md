# Identity Wallet Mobile App (CLAUDE.md)

## Summary

Comprehensive architecture documentation for the [[entities/identity-wallet|Identity Wallet]] — a Tauri v2 iOS application with SvelteKit 2 + Svelte 5 frontend and Rust backend. The frontend is a fully static SPA (SSR disabled) loaded by WKWebView, communicating with the Rust backend exclusively through Tauri's IPC bridge.

The frontend exposes typed IPC wrappers via `src/lib/ipc.ts` and implements two main flows: **Create** (mode_select → relay_config → welcome → ... → home) and **Import** (mode_select → identity_input → pds_auth → ... → home). The Rust backend provides Tauri IPC commands for account creation, device key management, DID ceremony, OAuth flows, identity management, PLC monitoring, and recovery.

Key architectural decisions: `#[cfg]`-based device key dispatch (software P-256 on macOS/simulator, Secure Enclave on real iOS), OAuth 2.0 with PKCE + DPoP, per-DID Keychain namespacing for multi-identity support, background PLC monitoring every 15 minutes, and graceful degradation on network errors.

## Key Takeaways

- **Tauri v2 architecture**: SvelteKit 2 + Svelte 5 frontend in WKWebView, Rust backend via IPC bridge. SSR disabled globally. Frontend dist loaded from disk.
- **Device key dispatch**: `#[cfg]`-based compile-time path selection — software P-256 on macOS/simulator, Secure Enclave on real iOS. Same public API (`get_or_create`, `sign`).
- **OAuth 2.0 + DPoP + PKCE**: Full OAuth flow with PAR, Safari redirect, deep-link callback, DPoP-bound tokens, and transparent lazy refresh (<60s remaining).
- **Multi-identity via Keychain namespacing**: Per-DID Keychain accounts (`"{did}:device-key"`) with a top-level `"managed-dids"` JSON array index. Device keys lazily generated.
- **PLC monitoring**: Background `tokio::time::interval` every 15 minutes + foreground `visibilitychange` listener. Graceful degradation on network errors (returns empty vec, logs warning).
- **Recovery override**: 72-hour window to counter unauthorized PLC operations. Builds signed counter-operation restoring pre-unauthorized state.
