# 🔴 Observation: Linear workflow: one issue per PR
User preference: work on one Linear issue at a time, open a PR for it, then move to the next. Don't batch multiple issues into a single branch/PR unless they're tightly coupled (like MM-107 blob storage + MM-108 uploadBlob which is the storage consumer). Push branches to the Tangled remote (knot.malpercio.dev) so PRs can be opened.
