- **V028**: Adds the `repo_seq` table — the **persistent firehose event log** backing
  `com.atproto.sync.subscribeRepos`. The sequencer (`firehose.rs`) was in-memory only, so `seq`
  reset to 0 and the replay backlog emptied on every restart/redeploy. Each sequenced event (a
  `#commit`, `#account`, `#identity`, or `#sync` frame) is now persisted here *before* being broadcast, so `seq` is
  monotonic across restarts (loaded from `MAX(seq)` on boot) and cursor replay reads missed events
  back out of the log. `seq` is an explicit `INTEGER PRIMARY KEY` (rowid alias for index-backed
  `seq > ?` scans) assigned by the in-process sequencer — **not** `AUTOINCREMENT` — so a failed
  insert doesn't consume a number and leave a hole in the dense prefix replay relies on. `event` is
  the DAG-CBOR payload that reconstructs the wire frame, including the commit's CARv1 `blocks`
  (stored because post-commit GC may have reclaimed the blocks the diff was built from); the record
  `value` is not stored (not on the wire). Append-only at the writer, but bounded by the
  `firehose_gc` retention sweep (`run_firehose_gc` prunes a contiguous prefix `seq ≤ watermark`,
  never the live frontier; `retained_boundary_seq`/`prune_below` do the SQL). Query functions live in
  `firehose_seq.rs`.
- **V027**: Adds the `transfers` table for planned device-swap sessions (`POST /v1/transfer/initiate` and the later accept/complete steps). State machine `pending → accepted → completing → complete`, plus a terminal `expired`. One active transfer per account is enforced by a partial UNIQUE index `idx_transfers_active_did ON transfers(did) WHERE status IN ('pending','accepted','completing')`. Expiry is **not** in the index predicate (SQLite requires deterministic predicates; `datetime('now')` is not), so `transfers.rs::insert_transfer` sweeps any expired active row to `expired` before inserting — a still-active row then trips the index and is reported as `DuplicateActive` (the 409 path). A second partial UNIQUE index `idx_transfers_active_code ON transfers(code) WHERE status IN ('pending','accepted','completing')` keeps the 6-char code (the accept-time lookup key) unique among active transfers; an INSERT that trips it is reported as `CodeCollision`, and `routes/transfer_initiate.rs` regenerates the code and retries (bounded). The two cases are told apart via `unique_violation_column`. FK→`accounts(did)`.

## Purpose
Owns SQLite connection lifecycle and schema migration for the PDS's server-level database.
Keeps database concerns out of handler code and provides a reusable pool+migration API
that can later serve per-user SQLite databases (Wave 3/4).

## Contracts
- **Exposes**: `open_pool(url: &str) -> Result<SqlitePool, DbError>`, `run_migrations(pool: &SqlitePool) -> Result<(), DbError>`, `DbError`
- **Guarantees**: Pool uses WAL journal mode and max 1 connection. The schema_migrations bootstrap DDL runs outside any transaction. Pending migrations and their bookkeeping inserts run inside a single transaction per call. Migrations are forward-only and idempotent. `schema_migrations` table tracks applied versions.
- **Expects**: Valid sqlx SQLite URL (e.g. `"sqlite::memory:"`, `"sqlite:path.db"`). Pool must be open before calling `run_migrations`.

## Dependencies
- **Uses**: sqlx (SqlitePool, SqliteConnectOptions), thiserror
- **Used by**: `main.rs` (startup sequence), `app::AppState` (holds the pool), test helpers
- **Boundary**: Handlers never call `open_pool`/`run_migrations` directly; they receive the pool via `AppState.db`

## Key Decisions
- Custom migration runner over sqlx's built-in `migrate!()`: gives control over transaction boundaries and avoids sqlx's `_sqlx_migrations` table
- Single connection pool (`max_connections(1)`): avoids SQLite write contention, sufficient for v0.1
- `open_pool` accepts `&str` (not Config/AppState): keeps the function reusable for future per-user DBs

## Invariants
- Migration SQL files are append-only; never modify an applied migration
- Migration versions are sequential positive integers starting at 1
- WAL mode is always enabled (set via SqliteConnectOptions, not raw PRAGMA)
- Foreign key enforcement is always on (set via SqliteConnectOptions .foreign_keys(true), not raw PRAGMA)
