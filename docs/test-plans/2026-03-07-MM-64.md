## Prerequisites

- macOS machine with Nix installed (multi-user or single-user)
- The `ezpds` repository cloned and checked out at the MM-64 branch (commit `1cc5cbb` or later)
- `direnv` installed and hooked into the user's shell (for HV-3)
- Access to a Linux x86_64 machine or CI runner with Nix installed (for HV-1)
- `rustup` installed outside of Nix (for HV-2)
- All automated operational tests passing

**Note:** All `nix develop` commands require the `--impure` flag due to devenv's requirement to access `$PWD`. Without it, devenv fails with "devenv was not able to determine the current directory".

## End-to-End: Fresh Clone to Working Dev Shell

**Purpose:** Validates that a new contributor can go from `git clone` to a fully working development environment with zero manual dependency installation beyond Nix itself.

1. Clone the repository to a temporary directory: `git clone <repo-url> /tmp/ezpds-test && cd /tmp/ezpds-test && git checkout MM-64`
2. Run `nix develop --impure --command bash -c 'rustc --version && cargo --version && just --version && sqlite3 --version'`
3. **Expected:** First run may take several minutes (downloading Nix store paths). All four commands succeed. The Rust version is stable.
4. Run `nix develop --impure --command cargo check` from the repo root.
5. **Expected:** Cargo resolves dependencies and type-checks the workspace. Exit code 0. Confirms the toolchain, sqlite3 headers, and pkg-config all integrate correctly for an actual build.
6. Clean up: `rm -rf /tmp/ezpds-test`

## End-to-End: direnv Integration

**Purpose:** Validates that `.envrc` + `use flake` provides automatic shell activation when entering the project directory.

1. Ensure direnv is installed: `direnv version` (should print a version).
2. Ensure direnv is hooked into your shell. For zsh, verify `eval "$(direnv hook zsh)"` is in `~/.zshrc`. For bash, verify `eval "$(direnv hook bash)"` is in `~/.bashrc`.
3. Navigate to repo root: `cd /path/to/ezpds`
4. Run `direnv allow` to approve the `.envrc` file.
5. Navigate away: `cd /tmp`
6. Navigate back: `cd /path/to/ezpds`
7. **Expected:** direnv prints loading/activation messages. After activation completes, run `which rustc` — it should point to a path inside `/nix/store/`, not a system rustc.
8. Run `rustc --version` — should match the stable version seen in Phase 2 tests.
